# Parallax ZachD-Style Cutaway Tutor Design

**Date:** 2026-06-09
**Event:** SuperAI NEXT Hackathon 2026
**Project:** Parallax

## One-Line Pitch

Parallax is a live AI tutor that researches machines and turns them into ZachD-style interactive 3D cutaway explainers: animated internal parts, voice Q&A, component selection, and adaptive replay when the learner misunderstands.

## Core Positioning

Parallax should not compete as a generic “AI 3D visualization generator.” The stronger wedge is:

> **AI-generated teaching choreography for realistic interactive machine cutaways.**

The user sees a polished procedural jet-engine cutaway. A Lesson Compiler agent researches the mechanism and generates the lesson plan, narration, quiz, and re-teach path. A Runtime Tutor agent then stays aware of the currently selected component while the learner asks questions.

The ZachD-style feel comes from cinematic cutaways, internal motion, exploded views, part isolation, labels, arrows, airflow particles, slow-motion replay, and clear cause/effect animation.

## Target Demo Memory

After 90 seconds, judges should remember:

> **The AI researches a machine and generates the explainer.**

Secondary memory:

> **You can select moving parts and ask a live tutor about exactly what you are touching.**

## MVP Scope

### Hero Machine

The MVP supports one polished machine template:

- `jet_engine`

The template represents a simplified turbofan/turbojet-style cutaway with these components:

- Fan
- Compressor
- Combustor
- Turbine
- Central shaft
- Nozzle
- Cutaway casing
- Airflow particles

### Generated By AI

The AI generates the lesson layer, not the raw 3D model:

- Lesson outline
- Step-by-step narration
- Component focus sequence
- Quiz question
- Misconception diagnosis
- Re-teach script
- Source summary from Exa
- Activity log messages

### Critical Hackathon Feature: Lesson Compiler v1

The hackathon demo must show that Parallax can generate a lesson plan from 0 to 1 for the fixed `jet_engine` template. This is critical to the agentic story. The 3D rig remains prebuilt, but the teaching artifact is compiled by the AI.

Lesson Compiler v1 runs as a controlled pipeline:

1. User requests a jet-engine explainer or clicks `Refresh with Exa`.
2. Compiler retrieves current sources through Exa.
3. Compiler extracts mechanism stages, component roles, and cause/effect links.
4. Compiler maps those stages to known template components and allowed animations.
5. Compiler generates validated lesson JSON.
6. App stores the result in cache and starts the lesson.

This should be visible in the activity log as a real agent workflow, not hidden behind a loading spinner.

### Prebuilt By The App

The app owns deterministic rendering and interaction:

- Procedural 3D jet-engine rig
- Component positions
- Component hitboxes
- Camera presets
- Allowed animations
- Explode vectors
- Isolate/dim behavior
- Replay segments
- Mouse/touch selection
- MediaPipe event mapping if enabled

This split is important: the AI never emits Three.js code. It emits validated lesson JSON that the renderer interprets.

## User Experience

### Screen Layout

Use a two-panel layout:

- Left: large 3D canvas with the jet-engine cutaway.
- Right: agent panel with selected component, push-to-talk button, transcript, activity log, Exa sources, quiz state, and fallback/cache status.

### Interaction Model

Component selection only changes visual focus. It does not make the agent speak automatically.

If the learner selects the compressor and asks “Why is this important?”, the agent receives:

```json
{
  "userQuestion": "Why is this important?",
  "selectedComponent": "compressor",
  "currentLessonStage": "compression"
}
```

The answer should be about the selected component in the current lesson context.

### Required 3D Interactions

For each major component, support:

- Select
- Highlight
- Isolate
- Explode
- Replay relevant segment

The minimum components requiring this behavior are:

- Fan
- Compressor
- Combustor
- Turbine
- Nozzle

The shaft is also important for the misconception demo.

### Voice Scope

Use a hybrid voice design:

- MVP implementation: push-to-talk.
- Product feel: live co-pilot.
- Future direction: always-listening interruption.

The app should remain usable through text input if microphone permissions fail.

### MediaPipe Scope

MediaPipe is secondary, not critical path.

Mouse/touch selection is the reliable input. MediaPipe hand tracking feeds the same event system:

```json
{
  "type": "component_selected",
  "source": "hand",
  "componentId": "compressor",
  "gesture": "pinch",
  "confidence": 0.86
}
```

If MediaPipe is unstable, the recording can use mouse input while still showing the design supports webcam hand interaction.

Use Exa for:

Use AWS for:

- S3 cached lesson JSON
- S3 cached generated narration audio if used

## Agent Architecture

Parallax has two agent layers. Keeping them separate prevents the live tutor from becoming slow while still letting the product claim real 0-to-1 lesson generation.

### Runtime Tutor Agent

The Runtime Tutor Agent is the low-latency agent used during the lesson. It should be built with Vercel AI SDK because it is close to the Next.js UI and needs streaming, structured responses, and renderer commands.

Responsibilities:

- Answer selected-component questions.
- Use `selectedComponentId`, `currentStepId`, and lesson context.
- Return short spoken/text answers.
- Suggest renderer commands such as `focusComponents`, `setExploded`, or `startReteach`.
- Diagnose quiz answers and trigger targeted replay.

### Lesson Compiler Agent

The Lesson Compiler Agent creates the teaching artifact before the lesson starts. For the hackathon MVP, this can be a controlled pipeline implemented in a Next.js API route with Vercel AI SDK structured output. For the stretch version, it can become a long-running multi-agent workflow using OpenAI Agents SDK, LangGraph, or a similar orchestration framework.

Responsibilities:

- Retrieve sources through Exa.
- Extract mechanism stages and component roles.
- Map real mechanism stages to the available `jet_engine` template.
- Generate lesson JSON, quiz, misconception map, and re-teach script.
- Validate output with Zod.
- Store successful artifacts in S3 or local cache.

### Stretch: Generalized Multi-Agent Compiler

The stretch architecture expands the Lesson Compiler into several specialist agents:

| Agent | Job |
|---|---|
| Research Agent | Uses Exa to collect and summarize reliable sources |
| Mechanism Analyst | Extracts parts, flows, stages, and cause/effect links |
| Visual Director | Maps concepts to available 3D templates, components, camera presets, and animations |
| Lesson Planner | Produces the teaching sequence and narration arc |
| Quiz Agent | Writes misconception-based checks and re-teach plans |
| Critic / Validator | Checks accuracy, schema validity, and template compatibility |

This stretch compiler should run asynchronously and store artifacts, not block the live tutor.

## Agent Loop

The agentic loop should be visible in the right panel:

1. **Perceive:** User selects component, asks voice/text question, or requests lesson compilation.
2. **Retrieve:** Lesson Compiler calls Exa for source-grounded mechanism info.
3. **Plan:** Compiler generates lesson sequence; Runtime Tutor generates answer or re-teach strategy.
4. **Act:** App focuses component, replays segment, isolates parts, or asks quiz.
5. **Observe:** User answer or selected component updates context.
6. **Recover:** On wrong answer or invalid compiler output, agent diagnoses, retries, falls back, or triggers targeted replay.

## Lesson JSON Contract

The agent returns validated JSON. Initial schema:

```json
{
  "topic": "Jet engine",
  "template": "jet_engine",
  "title": "How a Jet Engine Turns Air Into Thrust",
  "summary": "A jet engine compresses air, burns fuel, extracts turbine work, and accelerates exhaust.",
  "steps": [
    {
      "id": "intake",
      "focusComponent": "fan",
      "cameraPreset": "front_cutaway",
      "animation": "airflow_intake",
      "narration": "Air enters through the front fan and begins moving toward the compressor."
    }
  ],
  "quiz": {
    "question": "What powers the compressor stages?",
    "options": [
      { "id": "a", "text": "The front fan directly pushes them" },
      { "id": "b", "text": "The turbine spins a shared shaft" },
      { "id": "c", "text": "The nozzle pulls them forward" }
    ],
    "correctOptionId": "b",
    "misconceptions": {
      "a": "fan_powers_compressor",
      "c": "nozzle_powers_compressor"
    },
    "reteach": {
      "misconception": "fan_powers_compressor",
      "focusComponents": ["compressor", "shaft", "turbine"],
      "animation": "turbine_shaft_compressor_replay",
      "narration": "The compressor is driven by a shaft connected to the turbine. Hot exhaust spins the turbine, the turbine spins the shaft, and the shaft spins the compressor."
    }
  },
  "sources": [
    {
      "title": "Source title",
      "url": "https://example.com",
      "summary": "Short source note"
    }
  ]
}
```

Allowed component IDs:

- `fan`
- `compressor`
- `combustor`
- `turbine`
- `shaft`
- `nozzle`
- `casing`

Allowed camera presets:

- `wide_cutaway`
- `front_cutaway`
- `compressor_focus`
- `combustor_focus`
- `turbine_focus`
- `shaft_focus`
- `exhaust_focus`

Allowed animations:

- `idle_spin`
- `airflow_intake`
- `compression`
- `combustion`
- `turbine_drive`
- `exhaust_thrust`
- `full_sequence`
- `turbine_shaft_compressor_replay`

## Misconception Demo

The primary adaptive moment:

- Quiz: “What powers the compressor stages?”
- Wrong answer: learner confuses compressor/turbine power flow.
- Diagnosis: learner missed the shaft link between turbine and compressor.
- Re-teach: isolate compressor, shaft, and turbine; dim the rest; show hot gas spinning turbine, turbine spinning shaft, shaft spinning compressor.

This is the clearest proof that Parallax adapts the explainer, not just the text.

## Visual Style

Aim for realistic educational machinery, not photoreal CAD.

Key visual choices:

- Dark neutral background
- Brushed metal materials
- Transparent cutaway casing
- Warm glow in combustor
- Blue airflow particles before combustion
- Orange exhaust particles after combustion
- Component labels anchored in 3D
- Exploded view offsets along the engine’s x-axis
- Cinematic camera easing

## Non-Goals

The MVP will not include:

- Full arbitrary 3D model generation
- Blender/CAD pipeline
- Open-world navigation
- Always-listening voice
- Stripe monetization
- Multiple polished machine templates
- Generalized long-running multi-agent compiler for arbitrary new machines
- Scientific simulation accuracy

## Fallbacks

| Risk | Fallback |
|---|---|
| Exa fails or is slow | Use cached source summary and lesson JSON |
| LLM returns invalid JSON | Retry once, then load cached hero lesson |
| Microphone fails | Text input drives the same agent loop |
| TTS fails | Browser speech synthesis or text transcript only |
| MediaPipe is noisy | Mouse/touch remains the primary interaction |
| 3D performance drops | Reduce particle count and turn off post-processing |
| Procedural model looks too simple | Lean on lighting, labels, cutaway, and animation clarity |

## Judging Alignment

| Criterion | How Parallax Shows It |
|---|---|
| Agent overview | A Lesson Compiler agent researches and compiles the explainer; a Runtime Tutor agent teaches, answers, quizzes, and re-teaches |
| Autonomy | Compiler decides lesson sequence and source summary; Runtime Tutor decides answer strategy and re-teach path |
| Tool use | Exa retrieval, Vercel AI calls, AWS cache/TTS fallback |
| Human-in-loop | User selects components, asks voice questions, answers quiz |
| Failure handling | Cached lesson fallback, JSON validation, retry path, text fallback, mouse fallback |
| Demo quality | Cinematic jet-engine cutaway with visible agent trace |

## Stage Recording Script

1. Show Parallax loaded with jet-engine cutaway.
2. Click “Compile with Exa”; activity log shows source retrieval, mechanism analysis, template mapping, JSON validation, and cache write.
3. Lesson Compiler generates the lesson plan and starts sequence.
4. Engine animates intake, compressor, combustor, turbine, exhaust.
5. User selects compressor and asks: “Why is this part important?”
6. Agent answers using selected component context.
7. Quiz appears: “What powers the compressor stages?”
8. User intentionally answers wrong.
9. Agent diagnoses missing shaft link.
10. Scene isolates compressor, shaft, turbine and replays power flow.
11. End on line: “Parallax does not just show a model. It teaches the moving mechanism, watches where you break, and replays the exact internal cause.”
